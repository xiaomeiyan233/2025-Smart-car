# 2025-Smart-car
# Dudaohang-Haust
2025智能车竞赛-完全模型组国二方案  
Dataset是数据集，在官方数据集基础上增加了一些数据量，数据集格式为VOC  
ICAR为上位机代码，运行在Edgeboard上，在官方开源代码基础上修改了串口、重写环岛等所有元素、优化巡线、舵机改为模糊控制  
LQ_TC26xB_LIB_ADS1为下位机代码，主控为TC264，主要完成各模块驱动、上下位机通信  
比赛介绍:  
完全模型组比赛规则:https://blog.csdn.net/zhuoqingjoking97298/article/details/143958024  
  
方案介绍:  
1.上位机   
    (1)巡线方案：摄像头采集图像(320×240)后，完成图像预处理(逆透视变换、灰度化、高斯滤波、二值化)，得到二值化图像后自下向上逐行扫描每一行的像素值变化，找到每行白色有效区域的起始于结束位置，起始位置为道路左边缘，终止位置为道路右边缘，最终形成道路左右边缘的二维动态数组。  
    (2)控制方案：根据左右边缘数组，用贝塞尔曲线拟合中线，得到中线后，中线上每个点的横坐标加权求平均值得到controcenter，用横轴中点横坐标-controcenter得到横向误差，送到模糊控制器中算出角度的纠正值；通过中线上每个点坐标计算中线方差，利用方差对最高速度和最低速度进行线性插值，得到的效果为直道高速，转弯低速。  
    (3)环岛方案：状态机。根据道路左右边缘线的泄露等信息判断环岛；入环前根据特殊点切换状态；入环后陀螺仪进行水平角度积分，积累到一定角度之后切换状态出环；出环补线确保车辆运行平稳。 
    (4)元素方案：状态机。汉堡店元素类似于环岛，进汉堡店之前通过特殊点来切换状态，进入后通过陀螺仪积累角度切换状态；停车库元素：先判断左右车库；第一个状态由车库最下方特殊点位置进行切换，第二个状态为判断该进入哪个车库，同样通过特殊点判断，第三个状态为确定转弯的时机，同样根据特殊点的位置切换，第四个状态为入库，手动给道路做右边缘线补线，使车模沿指定方向行进，入库后进入下一个状态出库，同样补线出库，定时一段时间进入最后状态结束。  
    (5)模型：添加一些数据量进行训练，得出来的模型，综合指标要比官方给出来的好不少。
2.下位机  
    (1)基于龙邱库开发。  
    (2)电机增量式PID控速、ADC电池电压检测、串口多数据包通信(重要，上下位机有多个状态变量、转角以及速度需要传输)、陀螺仪姿态解算(只是简单加了死区来克服零漂、后续可以引入四元数，结合加速度计进行数据融合来抑制零漂，但实际测试下来，这种方案已经够用了)。

在轮胎不打滑的前提下，实测这套方案全元素有1.8m/，单独跑赛道大概2m/s   

# 上位机代码使用方法    
```C
板卡系统中终端进入build文件夹  
cmake ../src  
make icar  
./icar  
```
